HTML Interview Questions

Frequently asked HTML interview questions with detailed answers

HTML stands for HyperText Markup Language. It is the standard markup language used to create web pages. HTML describes the structure of a web page semantically and originally included cues for the appearance of the document. HTML elements are the building blocks of HTML pages, represented by tags.

HTML5 is the latest version of HTML with several improvements:
  • Semantic Elements: New elements like <header>, <nav>, <section>, <article>, <footer>
  • Multimedia Support: Native support for audio and video with <audio> and <video> tags
  • Graphics: Canvas and SVG for better graphics support
  • Forms: New input types like email, date, number, range
  • Storage: LocalStorage and SessionStorage
  • APIs: Geolocation, Drag and Drop, Web Workers

Semantic HTML elements are elements that clearly describe their meaning to both the browser and developer. Examples include:
  • <header> - Defines a header for a document or section
  • <nav> - Defines navigation links
  • <section> - Defines a section in a document
  • <article> - Defines independent, self-contained content
  • <aside> - Defines content aside from the main content
  • <footer> - Defines a footer for a document or section
  • <main> - Specifies the main content of a document
Benefits: Better SEO, improved accessibility, cleaner code structure.

The <!DOCTYPE html> declaration is an instruction to the web browser about what version of HTML the document is written in. In HTML5, it's simply:

<!DOCTYPE html>

This declaration ensures that the browser renders the page in standards mode, which is important for consistent rendering across different browsers.

Block-level elements:
  • Take up the full width available
  • Always start on a new line
  • Can contain other block-level and inline elements
  • Examples: <div>, <p>, <h1>-<h6>, <ul>, <li>, <form>
Inline elements:
  • Take up only as much width as necessary
  • Do not start on a new line
  • Can only contain other inline elements
  • Examples: <span>, <a>, <img>, <strong>, <em>

CSS Interview Questions

Common CSS interview questions with detailed explanations

CSS stands for Cascading Style Sheets. It is a stylesheet language used to describe the presentation of a document written in HTML or XML. CSS describes how elements should be rendered on screen, on paper, in speech, or on other media.

The CSS Box Model is a box that wraps around every HTML element. It consists of:
  • Content: The actual content of the box (text, images)
  • Padding: Clears area around the content (inside the border)
  • Border: Goes around the padding and content
  • Margin: Clears area outside the border
Total width = width + padding + border + margin

display: none:
  • Removes the element from the document flow
  • Takes up no space
  • Cannot be focused
  • Children are also not displayed
visibility: hidden:
  • Element is not visible but still takes up space
  • Remains in the document flow
  • Can be focused
  • Children can be made visible

Flexbox (Flexible Box Layout) is a CSS layout model that allows easy alignment, direction, order, and size of elements within a container. Key concepts:
  • Flex Container: Parent element with display: flex
  • Flex Items: Direct children of the flex container
  • Main Axis: Primary axis along which flex items are laid out
  • Cross Axis: Perpendicular to the main axis
Common properties: justify-content, align-items, flex-direction, flex-wrap, flex-grow, flex-shrink.

JavaScript Interview Questions

Essential JavaScript interview questions with detailed answers

JavaScript is a high-level, interpreted programming language that conforms to the ECMAScript specification. It's primarily used for web development but can also be used for server-side programming.

JavaScript Data Types:
  • Primitive Types: String, Number, Boolean, Undefined, Null, Symbol, BigInt
  • Reference Types: Object, Array, Function, Date, RegExp

var:
  • Function-scoped
  • Can be redeclared and updated
  • Hoisted to the top of its scope
  • Creates a property on the global object
let:
  • Block-scoped
  • Cannot be redeclared but can be updated
  • Not hoisted (TDZ - Temporal Dead Zone)
  • Doesn't create a global property
const:
  • Block-scoped
  • Cannot be redeclared or reassigned
  • Not hoisted (TDZ)
  • Must be initialized at declaration

A closure is a function that has access to variables in its outer (enclosing) scope even after the outer function has returned. Closures are created every time a function is created.

Example: 
function outerFunction(x) {
    return function innerFunction(y) {
        return x + y;
    };
}
const addFive = outerFunction(5);
console.log(addFive(3)); // Output: 8
Closures are used for data privacy, function factories, and callbacks.

== (Loose Equality):
  • Compares values after type coercion
  • May convert types to make comparison possible
  • Example: 5 == "5" // true
=== (Strict Equality):
  • Compares both value and type
  • No type coercion
  • Example: 5 === "5" // false
It's recommended to use === for predictable behavior.

React Interview Questions

Common React interview questions with comprehensive answers

React is a JavaScript library for building user interfaces, particularly single-page applications. It was developed by Facebook.

Main Features:
  • Component-Based: Build encapsulated components that manage their own state
  • Virtual DOM: Efficient updates through virtual DOM diffing
  • Unidirectional Data Flow: Data flows from parent to child components
  • JSX: Syntax extension for JavaScript
  • Hooks: State and lifecycle features in functional components
  • Large Ecosystem: Rich ecosystem of tools and libraries

The Virtual DOM is a JavaScript representation of the real DOM. It's a lightweight copy that React uses to optimize rendering performance.

How it works:
  1. State changes create a new Virtual DOM tree
  2. React compares the new tree with the previous one (diffing)
  3. Only the changed elements are updated in the real DOM
  4. This minimizes expensive DOM operations
Benefits: Better performance, batch updates, improved developer experience.

React Hooks are functions that let you use state and other React features in functional components.

Common Hooks:
  • useState: Manages component state
  • useEffect: Handles side effects
  • useContext: Consumes context values
  • useReducer: Alternative to useState for complex state
  • useMemo: Memoizes expensive calculations
  • useCallback: Memoizes functions
  • useRef: Accesses DOM elements or persists values

SQL Interview Questions

Essential SQL interview questions for database roles

SQL (Structured Query Language) is a domain-specific language used for managing and manipulating relational databases.

Main Categories:
  • DQL (Data Query Language): SELECT - retrieves data
  • DML (Data Manipulation Language): INSERT, UPDATE, DELETE - modifies data
  • DDL (Data Definition Language): CREATE, ALTER, DROP - defines structure
  • DCL (Data Control Language): GRANT, REVOKE - controls permissions
  • TCL (Transaction Control Language): COMMIT, ROLLBACK - manages transactions

INNER JOIN:
  • Returns only matching rows from both tables
  • Excludes non-matching rows
  • Most commonly used join type
OUTER JOIN:
  • Returns all rows from one table and matching from other
  • Includes non-matching rows with NULL values
  • Types: LEFT OUTER JOIN, RIGHT OUTER JOIN, FULL OUTER JOIN
Example: 
-- INNER JOIN
SELECT * FROM employees e
INNER JOIN departments d ON e.dept_id = d.id;

-- LEFT OUTER JOIN
SELECT * FROM employees e
LEFT JOIN departments d ON e.dept_id = d.id;

Database indexes are data structures that improve the speed of data retrieval operations on database tables.

Benefits:
  • Dramatically improves query performance
  • Reduces I/O operations
  • Enables faster data access
  • Helps with sorting and grouping operations
Trade-offs:
  • Increases storage requirements
  • Slows down INSERT, UPDATE, DELETE operations
  • Requires maintenance
Types: B-tree, Hash, Bitmap, Full-text indexes.

HR Interview Questions

Common HR interview questions and how to answer them

Best approach: Structure your answer in 3 parts:
  1. Present: Brief introduction about your current role/experience
  2. Past: Relevant experience and achievements that led to this point
  3. Future: Why you're interested in this role and company
Example: "I'm a software developer with 3 years of experience in web development. I've worked on various projects using React and Node.js, where I led a team to deliver a major e-commerce platform. I'm excited about this opportunity because it aligns with my passion for creating user-friendly applications and your company's innovative approach to technology."

For Strengths:
  • Choose strengths relevant to the job
  • Provide specific examples
  • Show how they benefit the company
For Weaknesses:
  • Be honest but strategic
  • Choose a real but manageable weakness
  • Show how you're working to improve it
  • Avoid clichés like "I'm a perfectionist"
Example for weakness: "I used to struggle with public speaking, but I've been actively working on it by joining Toastmasters and volunteering to present team meetings. I've seen significant improvement and now feel more confident presenting to groups."

Research the company and mention:
  • Company Values: How they align with your personal values
  • Products/Services: Genuine interest in what they do
  • Company Culture: What attracts you to their work environment
  • Growth Opportunities: How the role fits your career goals
  • Recent News: Show you're up-to-date with company developments
Example: "I've been following [Company]'s work in [specific area] and I'm impressed by your commitment to [company value]. Your focus on innovation and employee development aligns perfectly with my career goals. I believe my experience in [relevant skill] would contribute to your ongoing projects and I'm excited about the opportunity to grow with a forward-thinking company."

Freshers Interview Preparation

Essential tips and questions for fresh graduates

Preparation Strategy:
  1. Strengthen Fundamentals: Focus on core concepts of your primary language
  2. Data Structures & Algorithms: Practice common problems and patterns
  3. Projects: Have 2-3 good projects to discuss
  4. Mock Interviews: Practice with peers or online platforms
  5. Company Research: Understand the company's tech stack
  6. Problem Solving: Think aloud during technical questions
  7. Resume: Highlight relevant skills and projects

Recommended Projects:
  • Web Application: Full-stack project using frontend and backend
  • Mobile App: Native or cross-platform application
  • API Project: RESTful API with proper documentation
  • Database Project: Complex database design and queries
  • Open Source Contribution: Contribute to existing projects
Project Quality Checklist:
  • Complete and functional
  • Well-documented code
  • Deployed and accessible
  • Includes tests
  • Professional README
